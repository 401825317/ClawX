# PPT 视觉工作室

目标不是给文字套框，而是交付一份有叙事、有画面、每页构图服务于内容的真实 `.pptx`。

## 强制规则

- 默认用简体中文沟通；用户明确要求其他语言时再切换。
- `create_designed_pptx_file` 是新建高质量 PPT 的主工具。`create_pptx_file` 和 `{baseDir}/scripts/make-pptx.mjs` 仅是工具不可用或素材无法满足时的轻量兜底，使用兜底必须如实说明。
- 用户要求好看、高级、酷炫、发布会感、杂志感、图片化表达时，禁止用同一组文字卡片重复铺满全稿。
- 先完成叙事和视觉设计，再调用渲染工具。不要边写大纲边套固定模板。
- 只有 `.pptx` 已生成并通过工具质量门禁后才能结束；最终返回 `MEDIA:<absolute-path>` 或绝对路径。

## 工作流

1. 明确主题、受众、使用场景、页数和一句话视觉方向。没有指定时自行做合理选择，不为常规细节反问用户。
2. 建立逐页故事线：封面、冲突或机会、证据、核心主张、展开、行动或收束。标题应表达观点，不写“目录一、内容介绍、总结”等空标题。
3. 为每页指定一种主导画面：全幅图片、左右分屏、超大数字、数据图表、对比、时间线、人物或产品特写、表格、留白陈述。连续两页不得机械复用同一构图。
4. 获取真实视觉素材。优先使用用户提供的图片；否则用可用的图片生成工具生成主题素材，或用 `web_search` / `web_fetch` 找到可使用的公开图片并下载到工作区。产品、地点、人物和实物主题必须优先展示真实主体，不用模糊气氛图代替。
5. 按 `references/studio-schema.md` 组织 0-100 自由画布参数，调用 `create_designed_pptx_file`。所有图片必须先成为本地 PNG/JPEG/WebP 文件。
6. 根据工具返回的 `verification` 核对页数、图片/图表/表格数量、视觉页比例和布局签名数。门禁失败时调整构图后重试，不能直接切回同一模板。

## 视觉质量线

- 封面只保留标题、必要副标题和一个明确视觉焦点，不堆功能说明。
- 16:9；标题通常不小于 35pt，正文通常不小于 16pt，演示场景正文建议 20-24pt。
- 五页及以上时，至少 40% 的正文页包含图片、图表或表格，且至少出现 3 种明显不同的构图。
- 每页只承载一个主观点。长材料先压缩、拆页、图表化，不把整段文字缩小塞入页面。
- 图表要传达结论，不做无意义装饰；表格只保留观众需要比较的字段。
- 颜色来自题材、品牌或内容证据。不要把“换配色”当成设计变化，也不要默认紫色渐变科技风。
- 禁止文本互相遮挡、越界、重复标题、空页、占位文字、同一图片反复使用和卡片网格贯穿整稿。

## 已有 PPT 的美化

先读取原稿结构和素材，保留用户事实与必要品牌约束，再重建为新的不覆盖文件。复杂读取或转换使用 `office-toolkit`；最终仍优先由 `create_designed_pptx_file` 输出新版。

## 兜底

只有 `create_designed_pptx_file` 确实不存在或执行环境无法加载时，才使用 `create_pptx_file` 或本地 `make-pptx.mjs`。兜底稿必须保持正确页数和内容，最终明确说明它是基础版，不能把基础版说成高设计成品。
